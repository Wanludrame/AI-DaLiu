# 电子大刘 (AI-DaLiu)

AI 科幻创作与短片制作引擎，基于 [Claude Code](https://claude.ai/claude-code) Skills 构建。

## 这是什么

"电子大刘"是一套 Claude Code Skills，能够：

1. **以刘慈欣风格创作科幻小说** — 基于对大刘 26 部重要作品的逐篇精读与分析，学习其构思方法、叙事结构、场景写作和人物塑造的内核
2. **从文本自动生成 AI 短片** — 全流程自动化：文本分析 → 分镜脚本 → AI 图片/视频生成 → TTS 旁白 → 字幕 → BGM → 最终组装
3. **三体同人创作** — 在三体世界观内创作同人小说，严格遵守原作的物理规则和文明设定
4. **模拟大刘的日常生活** — 每天生成一篇"电子大刘日记"，模拟大刘在娘子关电厂的一天，为创作注入生活质感

## 电子大刘日记系统

电子大刘的作品缺乏"生活气"——大刘作品中那种来自中国基层工业小镇的烟火质感。为了解决这个问题，我们构建了一套日记模拟系统。

### 设计灵感：Stanford Generative Agents

架构参考了 Stanford 的 [Generative Agents: Interactive Simulacra of Human Behavior](https://arxiv.org/abs/2304.03442)（Park et al., 2023）。这篇论文用 LLM 驱动 25 个虚拟角色在沙盒小镇中生活，每个角色都能记住过去的经历、形成对其他角色的看法、自主规划日程。它的核心架构有三层：

1. **Memory Stream**（记忆流）：完整记录角色经历的时间线
2. **Reflection**（反思）：定期将记忆压缩为更高层次的认识
3. **Planning**（规划）：基于记忆和反思生成未来行为

我们的借鉴方式：

| Generative Agents 概念 | 电子大刘对应实现 |
|------------------------|-----------------|
| Memory Stream | `电子大刘日记/*.txt` — 每天一篇日记，构成完整的经历时间线 |
| Reflection | `电子大刘日记/state.md` — 对近期生活状态的压缩摘要，每天更新 |
| Planning | 日记生成流程中读取状态 + 近期日记，决定今天会发生什么 |
| Agent State | state.md 中的身体状况、情绪、人物动态、进行中事件线 |
| Social Memory | 固定人物表 + state.md 中每个人物的动态跟踪 |

关键区别：Generative Agents 模拟的是多角色交互，我们模拟的是**单一角色的内心生活**——重点不在社交网络，而在一个人的感官体验、阅读思考、和对世界的私人感受。这些内心活动最终渗透进科幻创作。

### 设定：平行时空

大刘仍然在山西阳泉娘子关发电厂做计算机工程师。电厂日常、小镇生活、同事家人都在，但世界是今天的世界——他通过新闻联播和手机接收 2026 年的真实新闻，用今天的信息量思考。

### 架构

```
每日流程：读取状态 → 搜新闻/天气 → 查阅近期日记 → 生成今日日记 → 更新状态 → 创作小说
```

**持久化状态文件** (`电子大刘日记/state.md`)：记录大刘的身体状况、情绪、在读的书、写作进度、电厂机组状态、每个人物的动态、进行中的跨天事件线。每次写日记前读取、写完后更新，确保生活的连贯性。

**固定人物表**：大刘的社交圈是固定的——主控室值班长老韩、热工班老搭档老张、新来的大学生小孙、食堂赵姐、妻子、女儿刘静、《科幻世界》杨编辑等。人物不会凭空出现或消失，人事变动必须有因有果。

**事件有后果**：感冒持续 3-7 天，吵架冷战 1-2 天，加班导致疲劳累积，同事请假就不会出现在当天日记里。所有状态变化通过 state.md 跨天延续。

### 融入创作

`/liu-cixin-writer` 在创作前会读取当天日记，将生活质感渗透进作品——不是照搬日记内容，而是让日记中的感官印记、情绪基调、思维碎片成为写作的土壤。
